# Data Generation Rules

## 사용자 생성

- 총 사용자: 1,000,000
- 나이:
  - 부모: 30~55세
  - 자녀: 5~25세

## 가족 구성

- 가족 수: 250,000
- 가족 크기: 2~8명
- 역할:
  - OWNER
  - PARENT
  - CHILD

## 데이터 공유 정책

- FIFO 또는 PRIORITY
- 가족 데이터 = 인원 × 5GB

## 정책 적용 확률

| 정책 | 확률 |
|------|------|
| 시간차단 | 15~30% |
| 서비스 차단 | 20~70% |

## 가족 신청/삭제 규칙

- 가족 신청 타입: `ADD | REMOVE | CREATE`
- 신청 대상자는 다건 처리 가능 (`family_apply_target`)
- `CREATE` 요청은 가족 대표가 새로운 가족 생성 목적의 신청으로 사용
- `REMOVE`는 즉시 반영하지 않고 `family_remove_schedule`에 등록 후 리셋 시점에 반영
- 더미 생성의 `CREATE` 신청은 비가족 사용자에서만 생성
- 더미 생성 건수:
  - `CREATE`: `5~10`건
  - `ADD`: `4~5`건
  - `REMOVE`: `4~5`건
- 더미 생성 상태: 모두 `PENDING`
- `CREATE` 대상자 수: 신청당 최소 2명(요청자 제외), 최대 8명
- `ADD` 대상자 수: 신청당 최소 1명, 가족 최대 인원(8명) 초과 금지
- `REMOVE` 대상자 수: 신청당 최소 1명, 삭제 후 가족 최소 인원(2명) 보장

## 정책 매핑 규칙

- 가족 대표(OWNER)가 가족 정책을 생성하며, 가족당 정책은 1~3건 생성
- 정책 생성 방식:
  - 관리자 템플릿 그대로 사용 (`COPY`, 새 `block_policy_id` 부여)
  - 관리자 템플릿 커스텀 사용 (`CUSTOMIZE`, 새 `block_policy_id` 부여)
  - 완전 신규 정책 생성 (`NEW`)
- 현재 더미 정책 타입은 운영 편의상 `SCHEDULED`만 생성 (`ONCE` 미생성)
- `policy_sub`는 해당 가족 정책(`block_policy_id`) 기준으로 가족 구성원별 적용 여부(`is_active`)를 기록

## 데이터 선물

- 50% 가족 발생
- 부모 → 자녀
- 최대 5GB
- 일 단위 요금제(`period=DAY`)는 제외
- 1.5GB 요금제(`amount=int(1.5*GB)`)는 제외
