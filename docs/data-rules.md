# Data Generation Rules

## 사용자 생성

- 총 사용자: 1,000,000
- 나이:
  - 부모: 30~55세
  - 자녀: 5~25세

## 가족 구성

- 가족 수: 250,000
- 가족 크기: 2~8명
- 역할:
  - OWNER
  - PARENT
  - CHILD

## 데이터 공유 정책

- FIFO 또는 PRIORITY
- 가족 데이터 = 인원 × 5GB

## 정책 적용 확률

| 정책 | 확률 |
|------|------|
| 시간차단 | 15~30% |
| 서비스 차단 | 20~70% |

## 가족 신청/삭제 규칙

- 가족 신청 타입: `ADD | REMOVE | CREATE`
- 신청 대상자는 다건 처리 가능 (`family_apply_target`)
- `CREATE` 요청은 가족 대표가 새로운 가족 생성 목적의 신청으로 사용
- `REMOVE`는 즉시 반영하지 않고 `family_remove_schedule`에 등록 후 리셋 시점에 반영
- 더미 생성의 `CREATE` 신청은 비가족 사용자에서만 생성
- 더미 생성 건수: `5~10`건
- 더미 생성 상태: 모두 `PENDING`
- 더미 대상자 수: 신청당 최소 2명(요청자 포함), 최대 3명

## 정책 매핑 규칙

- 더미 생성 시 템플릿(관리자) 정책을 그대로 써도 `block_policy`에 가족별 정책 row를 새로 생성
- `policy_sub`는 템플릿 ID가 아니라 새로 생성된 `block_policy_id`를 매핑

## 데이터 선물

- 50% 가족 발생
- 부모 → 자녀
- 최대 5GB
- 일 단위 요금제(`period=DAY`)는 제외
- 1.5GB 요금제(`amount=int(1.5*GB)`)는 제외
